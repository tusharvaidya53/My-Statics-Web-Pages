Event Creation
  • Multiple categories like sports, politics, stocks, crypto, global events etc. Platform will have standard categories and community can also decide to add.
  • Create Event under category. Example – US Presidential Election (We will consider this example for understanding throughout this document. Various prediction markers (with possible outcomes) will be defined under this event. e.g. who will win US Presidential election 2020 - 1. Biden 2. Trump 3. Other Difference vote shares 1. less than 5% 2. more than 5% 3. exact 5%
  • User can create an Event and Prediction market for the event. There will be some minimum amount first user will have to put to create event and prediction market. This market creator will get some extra benefit like some % of pool size. Community can decide and adjust amount and % of this mechanism.
Prediction on event outcome
There will be 2 types of prediction market
  1. Community Prediction Pool This is common prediction market where users can choose to join the community pool User will select prediction market (e.g. who will win US Presidential election 2020 ), user will select prediction (e.g. 1. Biden), decide on amount he want to put (e.g.100USD) For each option there will be collective amount,
    1. Biden – Total 1000 (User A-100 + User B 120 and so on)
    2. Trump – Total 900 (User C – 110 + User D 130 and so on)
    3. Other – Total 150 (User E – 10 and so on)
    In case of Biden winning, All users predicting option 1 (A, B, etc) will receive total amount 1000+900+150 = 2050 USD. Users from option 1 will share this amount proportional to the contribution they made in option1.
  2. Peer to Peer Prediction Market In this market 2 users will predict opposite to each other. There can be only 2 options in this prediction market. This option helps users to have better control on the returns he expect from his prediction. E. g. User A puts 100 USD, User B puts 100 USD. Winner will receive 200USD. (vs -in community pool one can not control the share in pool as many people join)
Outcome Resolution
  • While predicting on any event, user will have to contribute 2 amounts
    1. His share in the prediction market outcome.
    2. Arbitration Deposit – User will have to submit this till market outcome decided
  • Every event will have event resolution date while creating market. Event outcome to be decided within 3 days(or adjusted) of this.
  • Winning users can claim the winning pot. Like single user of multiple users from Option1 – Biden can claim the pool amount. If there is no objection from Option2 – Trump or Option 3 – Other within 3 days, market outcome is decided and user pool of Option1 gets all pool amount. Arbitration deposit will be returned to all users – whether winners or looser.
  • If there is objection (some minimum user % to raise objection e.g.5%), the prediction market is referred to Oracles and the arbitration deposit is withheld.
Dispute & Arbitration
  • When there is dispute amongst users for the outcome, market is referred to Oracles.
  • Oracles will vote about the outcome of the market within 3 days and the amount is distributed to winners.
  • Winners will receive arbitration deposit back. n Arbitration deposit from loosing users will be seized and distributed to Oracles eqully.
  • Oracle outcome will be decided by majority. (more than 70%). Oracles with wrong input will be flagged. 3 wrong input will disqualify Oracle from the system and Stake will be burned.
  • In the event of No oracle vote, market will be cancelled amount will be returned to users.
